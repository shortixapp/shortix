API reference

The Shortix REST API lets you create and manage short links programmatically. Pro plan required for API access.

Authentication

All API requests require a Bearer token in the Authorization header.

Authorization: Bearer sx_live_your_api_key_here

API keys are scoped to your account. Keep them secret — never expose them in client-side code or public repositories.

API keys

API key management is available in the Dashboard → Settings once your Pro subscription is active. Each key can be revoked independently. API key format: sx_live_<32 random chars>.

Keys are shown once on creation — store them immediately in your secrets manager.

POSTCreate a short link

POST /.netlify/functions/shorten
Content-Type: application/json
Authorization: Bearer sx_live_…

{
  "url": "https://example.com/very/long/path",
  "slug": "my-link"
}

The slug field is optional (Pro only). If omitted, a random 6-character code is generated. Only https:// and http:// URLs are accepted.

Response 200

{
  "id": "uuid",
  "slug": "my-link",
  "url": "https://example.com/very/long/path",
  "clicks": 0,
  "active": true,
  "created_at": "2026-06-28T10:00:00Z"
}

GETList your links

GET /.netlify/functions/links
Authorization: Bearer sx_live_…

Returns all links belonging to your account, ordered by creation date (newest first).

GETGet one link

GET /.netlify/functions/links/:id
Authorization: Bearer sx_live_…

The :id parameter can be either the UUID or the slug.

DELETEDelete a link

DELETE /.netlify/functions/links/:id
Authorization: Bearer sx_live_…

Immediately stops the short link from redirecting. This action is irreversible.
